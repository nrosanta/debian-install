Helps >
I've never needed to find out this info, as I seem to always be installing Debian on laptops.  But just now I installed on a desktop PC and the screen resolution was somewhat stuck on a low setting with no options in Preferences > Monitor Settings to set 1920x1080 pix.  

So, whether its a proprietry driver or its the kernel's driver that didn't detect the monitor's capabilities, we might need to add a config file to tell the system what res to turn on the screen with.

What we need to do is edit the "Monitor" and "Screen" sections of Xorg.conf - and I read that if there is no Xorg.conf file around (in /etc/X11 - and its not there any more by default as X loads up device settings as it sees them during boot) it can be generated with
cd /etc/X11
X11 -configure 
or 
X :1 -configure
(and that's when outside of any display manager - I exit Openbox first)

But I only got an error and no file made.

The next option is to create a file in /etc/X11/xorg.conf.d/ with just the Sections that we want Xorg to make the settings for,
sudo mkdir /etc/X11/xorg.conf.d
sudo leafpad /etc/X11/xorg.conf.d/10-monitor.conf

For my resolution requirement I put these lines in the file:
Section "Monitor"
Identifier   "Monitor0"
HorizSync    30.0 - 86.0
VertRefresh  56.0 - 76.0
Modeline     "1920x1080"  173.00  1920 2048 2248 2576  1080 1083 1088 1120 -hsync +vsync
Option    "PreferredMode" "1920x1080"
EndSection


Section "Screen"
Identifier "Screen0"
Device     "Device0"
Monitor    "Monitor0"

SubSection "Display"
Depth     24
Modes    "1920x1080"
EndSubSection
EndSection

you can add modelines to Monitor section and modes to Screen section, as you need.

You get the modeline by this command
cvt 1920 1080 60
ie cvt <width> <height> <refresh-rate>



If you are using a specific graphics card, then that might need adding too, with
Section "Device"
    Identifier    "Device0"
    Driver        "driver-name" #Choose the driver used for this monitor
EndSection


Detailed manual is here http://www.x.org/archive/current/doc/man/man5/xorg.conf.5.xhtml

A second monitor will need its own entries, and Monitor0, Screen0 and Device0 will change to Monitor1, Screen1 and Device1.
see https://gist.github.com/daemox/6325050
