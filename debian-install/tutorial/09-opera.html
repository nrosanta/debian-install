Here's a very good web-browser...  except that web page coding has quite advanced beyond it's engine's power, so they are rebuilding it with something like Chrome's browser engine.  Because of that, the latest version, 16, is still without lots of the normal features, after they had to rebuild from scratch, and that's not even released for Linux yet.

Already, Opera can save your session (menu > Tabs > Sessions) and work offline (menu > Settings >), it can also be your email manager, if you need it.  You can open a panels tab and there you can easily switch between bookmarks, notes, downloads, history, page links and page info. Also mail, contacts and chat if you set them up.  

Open the bookmarks in the panels tab with Ctrl-B, or history with Ctrl-H 
Ctrl-J opens downloads in another tab not in panels. :(

Editing bookmarks can be done in the panels tab (to import/export bookmark file, right click anywhere and File > ), or the Manage Bookmarks tab opened from Opera menu (Ctrl-Shift-B).

to add the bookmarks button to address bar (after you import html bookmark file from chrome/firefox):
  • hit Alt for Opera menu > Appearance (or Shift-f12) > Buttons tab > Browser
  • drag items to the address bar where you'd like them
  • you can drag any to a new position but to remove any you don't drag them off the bar, you right click and remove each one
I add the recently closed tabs button too.

Speed dial thumbs on a new tab page can be set to whatever pages and how many you want, and at what size you want.  

  • turn off special effects, Shift-f12
  • disable animated images, Ctrl-f12 > Content
  • disable remamber content on visited pages, Ctrl-f12 > History
  • set memory cache lower, and empty disk cache on exit, Ctrl-f12 > History
  • I got Opera down to 270mb, but it returned to 563mb with youtube (flash player plugin) plus 48mb operapluginwrapper.  Normal mem usage with about four tabs open can be 140 - 330mb, which is normal for a web browser -and it depends on how much content you open.

Some reasons why Opera is a better browser
  • turbo (f12) -designed for tablets, cuts out loading everything as each page gets filtered through opera-mini server -speeds up page loading on a slow connection (can't download from some pages with turbo on)
  • history is organised by website, not straight time-line
  • speed dial with shortcuts to web pages, customizable b-gd
  • mouse gestures (right click and hold for tips.. follow directions eg r-c down and up opens the page in background, ie duplicates it.  Keep moving before release right button and gesture is canceled.
  • inspect web page elements with dragonfly
  • highlight text and "copy to note"

Adapting the UI looks
1. There are two high-contrast options under Menu > Page > Style > , which are B/W and W/B
  • both these styles are easily configurable, as they sit in ~/.opera/styles/user/ - the files are contrastbw.css and contrastwb.css
  • the 1st "color:" and "background:" refer to the overall page (e.g. I set mine to black and grey)
  • the 3rd part is for links color and background (e/g I set mine to blue and silver)
  • this method destroys text colours from web sites however!

2. Set webpage colours the better way.
  1. Ctrl-f12 to open Preferences, go to Webpages tab and select background and font colours.
  2. Then go to Advance tab and select Content and Style Options.
  3. under Presentation Modes tab, uncheck "Page style sheet" and "Page fonts and colours" from both Author mode and User mode.
  4. make sure "My style sheet" and "My font and colors" is checked under both.
  5. But, without "Page style sheet" checked webpages look naff!

3. There's a setting you can play with called Dialog Toolkit -I don't really know what it does, but helpful if you can't read the font in the address bar.  Go to opera:config#FileSelector|DialogToolkit. Try setting to 3 or 4, save and restart.  see here http://help.opera.com/operaconfig/
  • 0 = Autodetect toolkit to use for file selector
  • 1 = Use Qt for file selector (deprecated, will fall back to KDE)
  • 2 = Use GTK for file selector
  • 3 = Use KDE for file selector
  • 4 = Use X11 for file selector

skinning 

1. work from existing skin in ~/.opera/skin/
2. unzip the skin
3. open the skin.ini file
4. search for attribute to edit
5. can use another skin with design you like, to help you

I found address field was in
[Edit Skin]
I added colors but can't make it change!
Text Color= #383838 

Notes text field is in
[MultilineEdit Skin]
I added colors with success!
Color= #323232
Text Color= #9BA3A4  

6. change name of skin at top of file and save it 
7. select all the contents and repackage to skin-name-edited.zip
8. move the new zip archive to the skin/ dir
9. restart Opera and select your skin from menu > Appearance
