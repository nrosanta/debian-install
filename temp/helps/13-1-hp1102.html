HP Printers

HP are kind to provide driver help for Linux systems, through hplipopensource.com
Hplip is in the Debian repo, and can be installed (if not already) with
sudo apt-get install cups cups-filters hplip

If you need the latest HPlip driver, download and run the install script from: http://hplipopensource.com/...
(and run the WIzard on the link at left to check what version you need to install - I'm not sure if one "does them all").
then cd to download directory, and run the script with,
sh hplip*.run
and follow the instructions

If the version of hplip installed with your debian version supports your printer type, then you should be able to add it in the CUPS Admin page:
  • connect the printer and turn it on
  • for hp printers (and others?) simply run 
hp-setup
  • this will locate your printer, download a tool to disable "smart-start" if necessary, and download any plugin required

  • For other printers, using CUPS, go to localhost:631/admin in a web browser
  • click on Add printer, login and follow the steps
  • if you can't get far in CUPS (error "Request entity too large"), see my printers page
  • if the printer shows hpcups failed, see below

HPlip Debian 8


Sanjay Kumar (sanjay-kumar14) said on 2015-02-21:
"Debian 8 is not yet supported in HPLIP automatic installer, however you can install HPLIP using manual instructions. Below are the steps.

=> Download and install the latest HPLIP release from
http://hplipopensource.com/hplip-web/install/manual/index.html (Use debian 7.x steps)
=> Reconfigure print queue using below commands.
=> hp-setup -r (remove all print queues)
=> su -c "hp-plugin"
=> hp-setup
=>Try printing now."



Trouble-shoot a hp printer that's not printing

run this command to check for errors, or needed dependencies:
hp-check -r

To sort out errors you can run
hp-doctor


Notes on installing a laserjet p1102

First of all, I installed the p1102 with CUPS and tried a test page
no print, reports: 
stopped
"/usr/lib/cups/filter/hpcups failed"
error log:
printer-state-reasons=hplip.plugin-error
installed the 3.12.6 plugin and still get error

  • So I ran hp-setup in a terminal, with this
hp-setup
  • selected USB connection
  • smart-start detected - downloads a fix
  • downloaded the plugin ppd file
  • printed the test page and it worked!

Make a panel launcher with
[Desktop Entry]
Name=P1102 Status
Comment=printer status
Exec=chromium http://localhost:631/printers/HP_P1102
Icon=printer
Terminal=false
Encoding=UTF-8
Type=Application
Categories=Settings;


Notes for installing hplip

  1. during hplip install, it prompts you to connect and turn on the printer 
  2. then it prompts for installing the plugin -so download it or locate it on disk
    I found the download link to the plugin file and put it into uget
    http://www.openprinting.org/download/printdriver/auxfiles/HP/plugins/hplip-x.yy.z-plugin.run
    there are other versions here at sourceforge http://hplip.sourceforge.net/plugin.conf
  3. actually, hp-setup will try to download it and it gives you the URL
    can also use command line to install plugin: sudo hp-plugin -i -g
    then     specify the path of the plugin .run file
    but downloading the digital signature might fail -install anyway 
  4. must add the user to the lpadmin group with sudo adduser <user> lpadmin


My P1102 had a horrible problem with an earlier hplip (3.12.4 I think) 
in hp device manager the printer had a communication error
In CUPS admin "/usr/lib/cups/backend/hp failed" in job state
CUPS  error log -it is clear that "backend" is not getting through to the printer
The soluton was a mode switch code!

Some new HP printers (WITH P1102 DEFINITELY) have smart install on their memory chip so that they can install onto a Windows machine (for ease of process) so on Linux the printer shows up as a usb CD Rom, and CUPS can't get it to print.
Eventually I found a ticket on Launchpad with a solution https://bugs.launchpad.net/hplip/+bug/672134 using usb_modeswitch 
sudo apt-get install usb-modeswitch
then I ran the code that Dudinea posted,
